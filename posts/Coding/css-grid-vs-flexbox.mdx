# CSS Grid vs Flexbox: When to Use What

CSS Grid and Flexbox are both powerful layout systems, but they solve different problems. Understanding when to use each can dramatically improve your CSS architecture and development workflow.

## The Fundamental Difference

**Flexbox is one-dimensional** - it handles layout in a single direction (row or column).
**Grid is two-dimensional** - it handles both rows and columns simultaneously.

```css
/* Flexbox - one direction at a time */
.flex-container {
  display: flex;
  flex-direction: row; /* or column */
}

/* Grid - both dimensions together */
.grid-container {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  grid-template-rows: auto 1fr auto;
}
```

## When to Use Flexbox

### 1. **Component-Level Layouts**

Flexbox excels at distributing space and aligning items within components:

```css
/* Navigation bar */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 1rem 2rem;
}

.nav-links {
  display: flex;
  gap: 2rem;
  list-style: none;
}

/* Card component */
.card {
  display: flex;
  flex-direction: column;
  padding: 1.5rem;
  height: 100%;
}

.card-content {
  flex: 1; /* Takes available space */
}

.card-actions {
  margin-top: auto; /* Pushes to bottom */
}
```

### 2. **Centering Content**

Flexbox makes centering trivial:

```css
/* Perfect centering */
.center-everything {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}

/* Center with text alignment */
.center-text {
  display: flex;
  flex-direction: column;
  align-items: center;
  text-align: center;
}
```

### 3. **Form Layouts**

Great for form inputs and button groups:

```css
/* Form row */
.form-row {
  display: flex;
  gap: 1rem;
  margin-bottom: 1rem;
}

.form-row > * {
  flex: 1; /* Equal width fields */
}

/* Button group */
.button-group {
  display: flex;
  gap: 0.5rem;
}

.button-group .btn {
  flex: 1;
}

/* Responsive form */
@media (max-width: 768px) {
  .form-row {
    flex-direction: column;
  }
}
```

### 4. **Media Objects**

Perfect for image + text combinations:

```css
.media-object {
  display: flex;
  gap: 1rem;
}

.media-object img {
  flex-shrink: 0; /* Prevent image from shrinking */
  width: 64px;
  height: 64px;
  border-radius: 50%;
}

.media-content {
  flex: 1; /* Take remaining space */
}
```

## When to Use CSS Grid

### 1. **Page-Level Layouts**

Grid is perfect for overall page structure:

```css
/* Classic website layout */
.page-layout {
  display: grid;
  grid-template-areas:
    "header header header"
    "sidebar main aside"
    "footer footer footer";
  grid-template-columns: 200px 1fr 150px;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
  gap: 1rem;
}

.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.aside { grid-area: aside; }
.footer { grid-area: footer; }

/* Responsive grid */
@media (max-width: 768px) {
  .page-layout {
    grid-template-areas:
      "header"
      "main"
      "sidebar"
      "aside"
      "footer";
    grid-template-columns: 1fr;
  }
}
```

### 2. **Card Grids**

Excellent for product grids, portfolios, galleries:

```css
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 2rem;
  padding: 2rem;
}

/* Masonry-style grid */
.masonry-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  grid-auto-rows: 10px;
  gap: 1rem;
}

.masonry-item {
  /* Heights determined by content */
  grid-row-end: span var(--row-span);
}
```

### 3. **Complex Dashboard Layouts**

Grid handles complex layouts with overlapping and spanning:

```css
.dashboard {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  grid-auto-rows: 100px;
  gap: 1rem;
}

.widget-large {
  grid-column: span 8;
  grid-row: span 3;
}

.widget-medium {
  grid-column: span 4;
  grid-row: span 2;
}

.widget-small {
  grid-column: span 3;
  grid-row: span 1;
}
```

### 4. **Magazine-Style Layouts**

Perfect for complex, magazine-like designs:

```css
.magazine-layout {
  display: grid;
  grid-template-columns: repeat(6, 1fr);
  grid-auto-rows: 100px;
  gap: 1rem;
}

.featured-article {
  grid-column: 1 / 5;
  grid-row: 1 / 4;
}

.secondary-article {
  grid-column: 5 / 7;
  grid-row: 1 / 3;
}

.small-articles {
  grid-column: 5 / 7;
  grid-row: 3 / 4;
  display: flex; /* Flexbox inside grid! */
  gap: 0.5rem;
}
```

## Combining Grid and Flexbox

The most powerful layouts use both together:

```css
/* Grid for page structure */
.page {
  display: grid;
  grid-template-areas:
    "header"
    "main"
    "footer";
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}

/* Flexbox for header components */
.header {
  grid-area: header;
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 1rem 2rem;
}

/* Grid for main content */
.main {
  grid-area: main;
  display: grid;
  grid-template-columns: 1fr 300px;
  gap: 2rem;
  padding: 2rem;
}

/* Flexbox for cards inside grid */
.card {
  display: flex;
  flex-direction: column;
  padding: 1.5rem;
  border: 1px solid #ddd;
  border-radius: 8px;
}
```

## Decision-Making Guidelines

### Choose Flexbox when:
- ✅ Working with components or small-scale layouts
- ✅ You need to distribute space between items
- ✅ You're aligning items in one dimension
- ✅ You need content-based sizing
- ✅ Building navigation, buttons, or form controls

### Choose Grid when:
- ✅ Creating page-level layouts
- ✅ You need precise control over rows and columns
- ✅ Working with overlapping elements
- ✅ Building card grids or galleries
- ✅ Creating responsive layouts with areas

### Use both when:
- ✅ Grid for page structure, Flexbox for components
- ✅ Building complex, responsive applications
- ✅ You need the best of both worlds

## Performance Considerations

```css
/* Avoid unnecessary nesting */
/* ❌ Over-nested */
.container {
  display: grid;
}

.container .row {
  display: flex;
}

.container .row .col {
  display: flex;
  flex-direction: column;
}

/* ✅ Simplified */
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1rem;
}

.container > * {
  display: flex;
  flex-direction: column;
}
```

## Browser Support

Both Grid and Flexbox have excellent modern browser support:

- **Flexbox**: 98%+ support (IE10+ with prefixes)
- **Grid**: 96%+ support (IE10+ with -ms- prefix)

```css
/* Progressive enhancement */
.layout {
  /* Flexbox fallback */
  display: flex;
  flex-wrap: wrap;
}

.layout > * {
  flex: 1 1 300px;
}

/* Grid enhancement */
@supports (display: grid) {
  .layout {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  }

  .layout > * {
    flex: none;
  }
}
```

## Conclusion

Grid and Flexbox aren't competitors—they're complementary tools:

- **Grid excels at 2D layouts**: page structure, complex grids
- **Flexbox excels at 1D layouts**: component alignment, space distribution
- **Together they're unstoppable**: use Grid for layout, Flexbox for components

Master both, understand their strengths, and choose the right tool for each job. Your CSS will be cleaner, more maintainable, and more flexible.

The future of CSS layout is bright, and with these tools in your toolkit, you can build any design you can imagine.
